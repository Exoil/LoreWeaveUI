# HTTP / API layer rules

The frontend talks to the backend through **three** layers — keep this split
strict:

1. `services/httpClients/LoreWeaveApiClient.ts` — **auto-generated by NSwag**
   from the backend OpenAPI contract (regenerate via
   `scripts/sync-api/sync-api.sh`; class name set in `loreweave-client.nswag`).
   Header at the top of the file says `<auto-generated>` — never hand-edit. To
   pick up backend contract changes, regenerate this file via the NSwag
   toolchain.
2. `services/LoreWeaveApiService.ts` — friendly wrapper that the rest of the
   app consumes. It:
   - Owns a single `LoreWeaveApiClient` instance.
   - Exposes `*Async` methods that accept `AbortSignal` and return
     domain models from `services/Models/` (not the generated DTOs).
   - Translates generated DTOs into `services/Models/*` classes.
3. Components / composables — call **only** `LoreWeaveApiService`. Never
   import `axios` or `LoreWeaveApiClient` directly.

When adding a new endpoint: regenerate `LoreWeaveApiClient`, add a wrapping
method on `LoreWeaveApiService`, and define any new `services/Models/*` types
that mirror the payload.

> Naming: `LoreWeaveApi*` is the **frontend's** API layer. The **backend** is a
> separate project still called RpgAssistant — use that name only when referring
> to the backend repo/contract in prose.
